# IAM-04: Privileged Identity Management (PIM)

## Objective
Minimize risk of excessive privileges by implementing Just-in-Time access.

---

## Configuration

- Role: Global Administrator
- Assignment Type: Eligible
- Activation:
  - MFA required
  - Justification required
  - Approval required
- Duration: 1 hour

---

## Risk

- Permanent admin privileges
- Insider threats
- Privilege abuse

---

## Control Mapping

- ISO 27001: A.9.2, A.9.4
- NIST: AC-2, AC-6
- CIS: Control 5

---

## Conclusion

PIM enforces least privilege by granting temporary administrative access only when required.
